## KLUCZOWE KOMPETENCJE

**Lokalne modele LLM (self-hosting / offline AI)**
- Stawianie i zarządzanie modelami uruchamianymi lokalnie przez **Ollama** (m.in. `qwen3:4b`) — pełny offline, bez kosztów API
- Integracja lokalnego modelu z edytorem przez **Continue.dev** w VS Code jako asystent kodowania
- Realne zarządzanie ograniczeniami sprzętu (GPU 4 GB VRAM): dobór rozmiaru modelu, throughput, tryby pracy
- Świadomy podział zadań: rutynowe → model lokalny; krytyczne (audyt, weryfikacja) → model chmurowy premium

**Systemy RAG i wyszukiwanie semantyczne**
- Zbudowany od zera **lokalny RAG offline**: embeddingi `bge-m3` + własny indeks wektorowy (numpy), pipeline ingest → search
- Pilotaż na realnym korpusie (~1300 fragmentów) z wysoką trafnością zapytań
- Dedykowane narzędzia do pracy z PDF (ekstrakcja, wyszukiwanie) zamiast ładowania całych dokumentów

**Automatyzacja i orkiestracja AI**
- Zaprojektowany **bramkowany pipeline badawczy** w stylu mini-działu R&D: generowanie → kodowanie → audyt → walidacja → raport, z automatycznymi bramkami jakości
- Praca z **MCP (Model Context Protocol)** — serwery narzędziowe rozszerzające możliwości asystentów AI
- Własny **system pamięci między sesjami** (pliki + synchronizacja), by narzędzie AI „pamiętało" wnioski z poprzedniej pracy
- Web scrapery do pozyskiwania danych z obsługą throttlingu i edge-case'ów

**Prompt engineering (zaawansowany, codzienny)**
- ChatGPT (OpenAI), Claude (Anthropic), Gemini (Google), Grok (xAI), DeepSeek, Qwen
- Iteracyjne dopracowywanie wyników, projektowanie wieloetapowych workflowów, instrukcje systemowe
- Generatywne obrazy: Midjourney, NanoBanana Pro — tworzenie i optymalizacja promptów wizualnych

## PROJEKT WŁASNY — System badawczy do analizy rynków

_Autorski, rozwijany długoterminowo · GitHub: github.com/33macioch33/portfolio_

Kompleksowy projekt, w którym pełnię rolę architekta i operatora sterującego narzędziami AI:

- **Pipeline badawczy**: automatyczny przepływ od hipotezy do raportu, z obowiązkowymi etapami weryfikacji (audyt danych, testy out-of-sample, symulacje Monte Carlo)

- **Lokalna infrastruktura AI**: modele offline + RAG + pamięć — ograniczenie zależności od płatnych API
- **Podejście „vibe coding"**: iteracyjne budowanie kodu z asystentami AI + krytyczna kontrola wyników
